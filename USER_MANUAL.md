## 系統簡介

MealWise AI 是一套智慧餐費結帳工具，能自動辨識餐費 PDF 報表中的員工資料與應收金額，並提供即時找零計算功能。

本系統採用 **BYOK（自備 API Key）** 架構，您的 API Key 僅儲存於您的本機瀏覽器中，不會傳送至任何第三方伺服器。

---

## 事前準備

### 申請 Google Gemini API Key

1. 前往 [Google AI Studio](https://aistudio.google.com/app/apikey)
2. 使用 Google 帳號登入
3. 點選「建立 API Key」
4. 複製產生的 API Key（格式為 `AIzaSy...`）

### 系統需求

- 現代瀏覽器（Chrome、Edge、Firefox、Safari 皆可）
- 網路連線（系統需連線至 Google Gemini API 進行 PDF 辨識）

## 操作流程

### 第一步：設定 API Key

首次開啟系統時，畫面會顯示 API Key 設定頁面。

1. 在輸入框中貼上您的 Gemini API Key
2. 點選「**進入系統並儲存 Key**」按鈕
3. 系統會將 Key 儲存至瀏覽器，下次開啟時無需重新輸入

### 第二步：上傳餐費 PDF

進入主畫面後，左側為上傳區域。

1. 點選「**選擇檔案**」按鈕
2. 選取餐費報表的 PDF 檔案（僅支援 PDF 格式）
3. 系統會自動開始辨識，畫面顯示「AI 正在解析報表...」

> **注意**：目前僅支援辨識 PDF 的**第一頁**內容。

### 第三步：查看辨識結果

辨識完成後，右側會顯示餐費明細表，包含以下欄位：

| 欄位 | 說明 |
|------|------|
| 排名 | 依應收金額由高到低排序 |
| 員工資訊 | 姓名與員工編號 |
| 應收金額 | AI 從報表中辨識出的餐費金額 |
| 已收金額 | 可手動輸入實際收到的金額 |
| 找零 / 結餘 | 自動計算（已收 − 應收） |

### 第四步：輸入已收金額

在「已收金額」欄位中輸入每位員工實際繳交的金額：

- 僅接受整數（0 至 50,000）
- 輸入後系統自動計算找零金額
- 綠色標籤表示「找零」（多收），紅色標籤表示「不足」（少收）

## 功能說明

### 總計資訊面板

位於畫面左下方，顯示：

- **本月應收總額**：所有員工應收金額的加總
- **目前實收總額**：已輸入金額的加總
- **完成進度條**：以百分比顯示收款進度

### 重新上傳

點選「**重新上傳**」連結可清除目前的辨識結果，重新上傳新的 PDF 檔案。

> **注意**：此操作會清除已儲存的辨識資料與已輸入的金額。

### 重設 API Key

點選右上角的「**重設 Key**」按鈕可清除已儲存的 API Key 並返回設定頁面。

> **注意**：此操作會同時清除所有已儲存的辨識資料。

## 資料保存機制

本系統使用瀏覽器的 **localStorage** 自動保存以下資料：

| 保存項目 | 說明 |
|----------|------|
| API Key | 儲存於 `mealwise_gemini_key` |
| 辨識結果 | 儲存於 `mealwise_parsed_data` |
| 檔案名稱 | 儲存於 `mealwise_file_name` |

### 保存特性

- 關閉瀏覽器或重新整理頁面後，資料**不會遺失**
- 已輸入的已收金額會**即時保存**
- 資料**無到期時間**，會一直保留直到手動清除

### 資料清除方式

以下操作會清除已保存的資料：

1. 點選「**重新上傳**」— 清除辨識結果與已輸入金額（保留 API Key）
2. 點選「**重設 Key**」— 清除所有資料（包含 API Key）
3. 手動清除瀏覽器的網站資料或快取

## 常見問題

### Q：出現「API Key 無效或額度不足」怎麼辦？

- 確認您的 API Key 是在 [Google AI Studio](https://aistudio.google.com/app/apikey) 申請的
- 確認 Key 未過期或被停用
- 檢查是否超出免費使用額度

### Q：辨識結果不正確怎麼辦？

- 確保 PDF 報表清晰可讀
- 目前僅支援辨識第一頁，請確認資料在第一頁
- 可點選「重新上傳」重新辨識

### Q：換一台電腦後資料還在嗎？

不在。資料儲存於當前瀏覽器的 localStorage 中，僅限同一台電腦的同一個瀏覽器存取。

### Q：手機可以使用嗎？

可以。系統支援手機瀏覽器，並提供專門的手機版卡片式介面。

### Q：多人可以同時使用嗎？

每位使用者需自備 API Key，各自在自己的瀏覽器中獨立運作，互不影響。
